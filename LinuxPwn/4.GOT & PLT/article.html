0x00 前言

无论是在ELF逆向以及Linux Pwn中，GOT和PLT这一对哥们都有着很重要的地位，比如前者的Hook，后者的结合任意写漏洞覆盖GOT表system()函数地址提权

调试一个Demo

GOT，全名Global Offset Table，中文俗称全局偏移表，注意这个全局

PLT，全名Procedure Linkage Table，中文俗称过程链接表，过程链接生动形象的描述了它的作用

这篇文章其实是我写的关于Linux Pwn系列文章的第四篇，有一些调试技巧可以参考前三篇，包括第三篇关于ELF文件格式

说多了都是瞎扯，我们直接写个小小的程序来使用gdb调试，观察具体的过程，我们观察printf()函数在两次调用时.got.plt的变化，中间的初始化数组只是缓一下节奏，跟第一个调用过程有一个对比

#include <stdio.h>
#include <string.h>

int main()
{
    char buffer[10];
    printf("I am Str one\n");
    memset(buffer, 0, strlen(buffer));
    printf("I am Str two\n");
    return 0;
}

0x02 文字形式的补充

前面这样的描述，有些地方还是难以说清楚，所以在这里再以个人经验描述一下

首先是.plt，从前面的代码可以比较清楚的看出，我们在调用printf()函数时，最先进入的就是printf()函数在.plt对应的代码段，然后.plt里对应的第一句指令是跳往.got.plt，在第一次调用printf()函数时，.got.plt对应的指令是跳回.plt，也就是说，所有需要重定位的函数在.got.plt都是一样的指令，在经过第一次调用后，.got.plt对应的地址就会写上printf()函数在内存中的绝对地址，在第二次调用printf()函数时，先跳到.plt，第一句指令跳到.got.plt，此时.got.plt已经写入printf()函数在内存中的绝对地址，直接调用即可

上面是一个大概的过程，讲了.plt与.got.plt的区别以及各自的调用顺序和作用

那么还有一个.rel.plt，这玩意就是我们在一开始看到的那个readelf -r，它存储着所有需要重定位函数的信息，而.got.plt是用于存储所有计算过后的函数内存地址

在Linux和Android下，都需要用到ELF，但是它们的链接器是不一样的，Linux使用ld.so，上面这一大堆说的就是Linux下的延迟绑定，但是在Android里，链接器是Linker，会一次性加载所有的函数地址，没有Linux下的延迟绑定
