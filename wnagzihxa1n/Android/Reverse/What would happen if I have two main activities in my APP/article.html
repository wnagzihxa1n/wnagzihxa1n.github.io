What would happen if I have two main activities in my APP

Author：wnagzihxa1n
Mail：tudouboom@163.com

0x00 前言

有时候会在一些病毒样本里看到AndroidManifest.xml有两个main activity：<action android:name="android.intent.action.MAIN"/>，有时候也会有两个LAUNCHER：<category android:name="android.intent.category.LAUNCHER"/>
分析病毒，对这些开发知识应该是要非常了解，才能对病毒的功能行为有一个很好的判断，当然不仅仅是在病毒分析方面

决定程序的入口Activity

<action android:name="android.intent.action.MAIN"/>

决定是否显示在程序列表里，直观点就是应用图标

<category android:name="android.intent.category.LAUNCHER"/>

0x01 two main activities

新建一个Demo，SDK使用4.4，系统6.0

初始化后MainActivity代码如下：

package com.wnagzihxain.demo;

import android.app.Activity;
import android.os.Bundle;

public class MainActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
    }
}

AndroidManifest.xml文件

<?xml version="1.0" encoding="utf-8"?>
<manifest package="com.wnagzihxain.demo"
          xmlns:android="http://schemas.android.com/apk/res/android">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
    </application>

</manifest>

我们创建一个新的Activity，命名为Second

}

将Activity：Second补充完整

import android.app.Activity;
import android.os.Bundle;

 * Created by wnagzihxain on 2017/1/13 0013.

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_second);
    }
}

准备工作做完后，我们开始配置AndroidManifest.xml文件

我们所有使用到的Activity都需要在AndroidManifest.xml文件进行注册

<?xml version="1.0" encoding="utf-8"?>
<manifest package="com.wnagzihxain.demo"
          xmlns:android="http://schemas.android.com/apk/res/android">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>

        <activity android:name=".Second">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
            </intent-filter>
        </activity>
    </application>

</manifest>

跑起来，发现进入的是第一个MainActivity，桌面上只有一个图标

咦？直接进入了第一个Activity，是不是和Activity在AndroidManifest.xml的注册顺序有关？

调换一下两个Activity的注册顺序

跑起来后进入的是MainActivity，这几轮下来我已经有点习惯这个结果了，但是发现桌面竟然有两个一模一样的图标

而且这两个图标依次点击进入的是不同的Activity，也就是说：两个Activity都创建了图标，并且可以通过点击进入

  • 当我们点击MainActivity创建的图标，我们会进入MainActivity
  • 当我们点击Second创建的图标，我们会进入Second

但是还有一个问题，这一次有两个入口，刚刚测试程序进入的是MainActivity，而不是Second，会和注册的顺序有关吗？

修改一下顺序

<?xml version="1.0" encoding="utf-8"?>
<manifest package="com.wnagzihxain.demo"
          xmlns:android="http://schemas.android.com/apk/res/android">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        <activity android:name=".Second">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>

        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
    </application>

</manifest>

跑起来进入的是Second
